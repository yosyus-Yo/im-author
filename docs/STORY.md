# AI Book Agent - 프로젝트 스토리

## 한 줄 요약
> "주제 하나를 입력하면, 4명의 AI 전문가가 실시간으로 협업하여 책을 만들어드립니다."

---

## 문제 (Problem)

책을 쓰는 것은 어렵습니다. 하지만 더 어려운 것은 **혼자서 모든 역할을 해야 한다는 것**입니다.

- 기획자: "이 주제로 책을 써도 될까? 시장이 있을까?"
- 작가: "어떤 구조로 써야 할까? 문장은 이게 맞나?"
- 편집자: "이 주장에 근거가 있나? 문법은 맞나? 독자가 이해할 수 있나?"
- 출판담당: "판형은? 가격은? 키워드는? 표지는?"

혼자서 이 네 가지 역할을 전환하면서 책을 쓰면 **6개월~1년**이 걸립니다.

---

## 해결 (Solution)

**AI Book Agent**는 이 4가지 역할을 각각 전문 AI 에이전트에게 맡깁니다.

```
사용자: "AI 시대의 글쓰기"

  🔍 리서처: "시장 분석 완료. 연 35% 성장 중, 경쟁 도서 3권 분석, GO 판정."

  ✍️ 작가: "7장 목차 설계 완료. 1장 '왜 AI와 함께 글을 쓰는가' 집필 중..."
         "자기편집: '수동태→능동태' 수정 3건, '모호한 표현' 수정 2건"

  📝 편집자: "7단계 검수 완료. 명확성 8점, 근거 6점."
            "수정 요청: 'GPT-4 출시 시점 구체화하라'"
            → 작가에게 피드백 전달 → 재작성 → 재검수 → APPROVED

  📦 출판담당: "메타데이터 생성, 표지 디자인 방향 결정, A5 판형 15,000원 책정."

결과: 주제 입력 → 시장 조사 → 목차 → 집필 → 편집 → 출판 준비 = 자동화
```

---

## 모든 장르 지원

| 장르 | 시놉시스 | 편집 기준 |
|------|---------|----------|
| 논픽션 | 핵심 주장 + 타겟 독자 | 근거 검증, 실용성 |
| 판타지 | 세계관 + 인물 + 갈등 | 세계관 일관성 |
| 로맨스 | 두 주인공 + 감정선 | 감정이입, 관계 발전 |
| 웹소설 | 능력 + 시스템 설정 | 클리프행어, 페이지 터너 |
| 수필 | 에피소드 + 테마 | 진정성, 공감 |
| 자서전 | 타임라인 + 교훈 | 장면 구체성 |
| 미스터리 | 사건 + 용의자 + 단서 | 페어플레이, 복선 회수 |
| SF | 과학 설정 + 사회 배경 | 내적 일관성 |

장르를 선택하면 모든 에이전트의 프롬프트가 자동으로 해당 장르에 맞게 조정됩니다.

---

## 차별점 (Why This is Different)

### 기존 AI 글쓰기 도구
```
사용자 → [하나의 AI] → 텍스트 출력
         (모든 역할을 하나가)
```
- ChatGPT에게 "책 써줘" → 품질 관리 없는 단순 생성
- 검수 없음, 시장 분석 없음, 자기편집 없음

### AI Book Agent
```
사용자 → [리서처] → [작가] ⇄ [편집자] → [출판담당]
          검증        창작    피드백 루프    포맷팅
```
- **역할 분리**: 각 에이전트가 독립적으로 판단
- **피드백 루프**: 편집자가 거절하면 작가가 재작성
- **자기편집**: 작가가 초안 후 3계층(구조→명확성→문체) 자가 검수
- **Go/No-Go**: 리서처가 안 되는 주제에 "NO"라고 말할 수 있음

## 핵심 데모 포인트 (심사위원에게 어필할 3가지)

### 1. "에이전트는 단순 텍스트 생성이 아니라, 판단을 합니다"
- 리서처의 Go/No-Go 판정 (confidence 87%)
- 편집자의 APPROVED/NEEDS_REVISION 판정

### 2. "에이전트끼리 실제로 대화합니다"
- 편집자 → 작가: "GPT-4 출시 시점을 구체화하세요"
- 작가 → 편집자: "수정했습니다. 재검수 부탁합니다"
- 실시간으로 채팅 패널에서 대화가 흘러감

### 3. "AI는 도구이고, 창작의 주인은 사용자입니다"
- 모든 단계에서 사용자가 프롬프트를 입력해 수정 가능
- 시놉시스, 목차, 챕터, 편집 결과, 출판 정보 — 어디서든 "이건 이렇게 바꿔줘"
- 사용자가 직접 쓴 문장은 AI가 절대 건드리지 않음
- AI가 도우미이지, 사용자를 대체하지 않음

### 4. "한 번의 입력으로 출판 준비까지 자동화됩니다"
- 주제 입력 → 시장 분석 → 목차 → 집필(스트리밍) → 편집(피드백 루프) → 출판 메타데이터
- 6단계 파이프라인이 시각적으로 진행

## 확장 가능성 (발표 마무리에 언급)

현재: 4 에이전트 (리서처, 작가, 편집자, 출판담당)

미래 (프로덕션):
- **온라인 출간 + 커뮤니티**: Supabase로 책 저장, 공개 출간, 웹 뷰어로 다른 사람 책 읽기
- **사용자 서재**: 로그인하여 본인이 만든 책 목록 관리
- **마케터 에이전트**: SNS 게시글, 광고 카피 자동 생성
- **AI 표지 생성**: DALL-E/Gemini로 실제 표지 이미지 생성
- **EPUB/PDF 변환**: 실제 전자책 파일 자동 생성
- **Self-Learning**: 에이전트가 매 작업에서 학습하여 품질 자동 향상
- **TurboQuant 최적화**: 로컬 LLM에서 전체 원고를 한 번에 처리 (메모리 6배 절감)
