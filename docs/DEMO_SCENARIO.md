# AI Book Agent - 데모 시나리오 & 발표 스크립트

> Build with TRAE @Seoul (2026-03-28)
> 발표 시간: 약 3~5분

---

## 데모 시나리오

### 1. 오프닝 (30초)

**발표 스크립트:**
> "안녕하세요. AI Book Agent를 소개합니다.
> 책을 쓰려면 기획, 집필, 편집이라는 긴 과정을 거쳐야 합니다.
> 저희는 이 과정을 3명의 AI 에이전트에게 맡겼습니다.
> 리서처, 작가, 편집자. 이 세 에이전트가 실시간으로 협업하여 책을 만드는 과정을 보여드리겠습니다."

### 2. 장르 선택 + 주제 입력 (30초)

**화면:** 장르 드롭다운 + 주제 입력란
**행동:** [논픽션] 선택 → "AI 시대의 글쓰기" 입력 → 시작 클릭

**발표 스크립트:**
> "먼저 장르를 선택합니다. 논픽션부터 판타지, 로맨스, 웹소설까지 8개 장르를 지원합니다. 주제를 입력하면 파이프라인이 시작됩니다."

### 3. 리서처 에이전트 작동 (30초)

**화면:** 왼쪽 채팅에 리서처 메시지, 오른쪽에 분석 결과
**포인트:**
- 시장 트렌드 분석 결과
- 경쟁 도서 2~3권 비교
- Go/No-Go 판정 (초록색 GO 배지)
- 신뢰도 87% 표시

**발표 스크립트:**
> "리서처가 시장을 분석하고 있습니다. AI 글쓰기 시장은 연 35% 성장 중이고, 멀티에이전트 출판을 다룬 책은 아직 없다고 판단했습니다. Go 판정이 나왔으니 다음 단계로 넘어갑니다."

### 4. 시놉시스 확인 (30초)

**화면:** 시놉시스 카드 (줄거리 + 인물 + 갈등 + 테마)
**포인트:**
- AI가 장르에 맞는 시놉시스 자동 생성
- 주요 인물/개념, 핵심 갈등, 테마 표시
- 하단 피드백 바에서 수정 가능

**발표 스크립트:**
> "리서처가 GO 판정을 내리면 작가가 시놉시스를 작성합니다. 사용자는 여기서 '이대로 진행'을 누르거나, 하단에 '주인공을 여자로 바꿔줘' 같은 수정 요청을 할 수 있습니다. 모든 단계에서 이 피드백 바로 사용자의 의견을 반영할 수 있습니다."

### 5. 작가 에이전트 작동 (40초)

**화면:** 목차 확정 → 1장 초안 스트리밍 작성 → 자기편집 로그
**포인트:**
- 7개 챕터 목차 자동 생성
- 1장 초안이 **실시간 스트리밍**으로 글자 하나씩 표시 (타이핑 효과)
- 글자 수 카운터가 실시간 증가
- 완료 후 "자기편집 과정 보기" 펼침 → 3계층 편집 기록 표시

**발표 스크립트:**
> "작가가 목차를 설계하고 1장을 실시간으로 집필합니다. 글자가 하나씩 나타나는 걸 보실 수 있죠. 그리고 여기, 작가의 자기편집 과정을 보시면 — 구조, 명확성, 문체 3계층으로 스스로 초안을 수정한 기록이 남아 있습니다. 사람 작가처럼 초안을 쓰고 스스로 다듬는 과정을 거칩니다."

### 6. 편집자 에이전트 작동 + 피드백 루프 (40초)

**화면:** 편집 검수 결과 → NEEDS_REVISION → 작가 재작성 → 재검수
**포인트:**
- **7가지 기준** 검수 (명확성, 근거, 구조, **문법**, 일관성, 몰입도, 교육성)
- 문법 체크에서 "됬다→됐다", "컨텐츠→콘텐츠" 등 교정
- 점수 시각화 (바 차트)
- NEEDS_REVISION 판정 → **피드백 루프 시작**
- 작가가 편집자 피드백을 반영하여 재작성
- 재검수 → APPROVED (점수 향상 확인)

**발표 스크립트:**
> "편집자가 7가지 기준으로 검수합니다. 문법 체크에서 '됬다'를 '됐다'로, '컨텐츠'를 '콘텐츠'로 교정하는 것도 보입니다. 그리고 중요한 포인트 — NEEDS_REVISION 판정이 나왔습니다. 이때 자동으로 작가 에이전트에게 피드백이 전달되고, 작가가 수정본을 다시 제출합니다. 편집자가 재검수해서 APPROVED가 나올 때까지 이 루프가 반복됩니다. 실제 출판사의 작가-편집자 관계와 동일합니다."

### 7. 출판담당 에이전트 작동 (20초)

**화면:** 출판 메타데이터, 표지 방향, 포맷 정보
**포인트:**
- 검색 최적화 키워드 자동 생성
- 표지 디자인 방향 (컬러 + 스타일 + AI 이미지 프롬프트)
- 판형/가격 자동 결정
- "출판 준비 완료" 배지

**발표 스크립트:**
> "편집이 끝나면 출판담당 에이전트가 출판에 필요한 모든 정보를 자동으로 준비합니다. 메타데이터, 표지 디자인 방향, 판형과 가격까지. 특히 표지는 AI 이미지 생성용 프롬프트까지 만들어줍니다."

### 8. 마무리 (30초)

**화면:** 완료 화면 (책 표지 + 통계)

**발표 스크립트:**
> "이렇게 주제 입력 하나로 시장 조사, 목차 설계, 집필, 편집, 출판 준비까지 자동화됩니다.
>
> 핵심은 단순 텍스트 생성이 아니라, 각 에이전트가 전문 역할을 갖고 독립적으로 판단한다는 것입니다. 리서처는 안 되는 주제에 No라고 말할 수 있고, 편집자는 작가의 글을 실제로 거절할 수 있습니다.
>
> 이 시스템을 확장하면 마케터(SNS 콘텐츠) 에이전트를 추가하고, 실제 EPUB/PDF 변환까지 연결하여 완전한 자동 출판 파이프라인을 구축할 수 있습니다.
>
> 감사합니다."

## 예상 Q&A

### Q1: "실제로 책 한 권을 완성할 수 있나요?"

> "현재 데모는 1챕터만 보여드리지만, 파이프라인 구조상 모든 챕터에 동일한 프로세스를 반복 적용할 수 있습니다. 실제 프로덕션 버전에서는 6개 에이전트가 전체 출판 과정을 담당합니다."

### Q2: "편집 에이전트의 검수 기준은 어떻게 정했나요?"

> "실제 출판사의 편집 체크리스트를 참고했습니다. 명확성, 근거, 구조, 일관성, 몰입도, 교육적 가치라는 6가지 축은 대형 출판사에서 사용하는 평가 기준을 축소한 것입니다."

### Q3: "다른 AI 모델로도 가능한가요?"

> "네, 시스템 프롬프트와 Function Calling 스키마가 핵심이므로, Claude나 Gemini 같은 다른 모델로도 교체 가능합니다. 중요한 건 에이전트의 역할 분리와 파이프라인 구조입니다."

### Q4: "에이전트끼리 의견이 다르면 어떻게 하나요?"

> "편집자가 MAJOR_REVISION을 내리면 작가에게 되돌려서 수정을 요청하는 피드백 루프가 있습니다. 이 루프를 최대 3회까지 반복한 후에도 기준에 미달하면 해당 챕터를 재작성합니다."

### Q5: "TRAE로 어떻게 만들었나요?"

> "사전에 에이전트 프롬프트, Function Calling 스키마, UI 컴포넌트 설계를 문서로 준비하고, TRAE에게 각 Phase별로 구체적인 지시를 내려 구현했습니다. 문서가 구체적일수록 TRAE의 코드 생성 정확도가 높아져서 3.5시간 안에 완성할 수 있었습니다."

### Q6: "기존 출판 시스템과 뭐가 다른가요?"

> "기존 AI 글쓰기 도구는 '하나의 AI가 모든 걸 하는' 구조입니다. 저희는 역할을 분리해서 리서처의 시장 분석, 작가의 창작, 편집자의 검수를 독립적으로 수행합니다. 마치 실제 출판사 팀처럼요."
